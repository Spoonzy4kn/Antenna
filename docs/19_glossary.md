# Glossary

**Project:** Antenna  
**Status:** Draft  
**Version:** 0.1  
**Last Updated:** 30 June 2026

---

## Terms

| Term | Meaning |
| --- | --- |
| Antenna | The open-source Android application defined by this project. |
| Anonymous user | A user who has not signed in to a platform account. |
| Authenticated user | A user who has signed in to a supported platform account. |
| Community | A public discussion area, such as a subreddit. |
| Content platform | A service that hosts community-driven discussion content. |
| Feed | An ordered list of posts. |
| Home | Primary feed screen, based on My Subs where supported. |
| Local favourite | A community saved locally on the device. |
| My Subs | User-facing name for local favourites. It does not necessarily mean platform-managed subscriptions. |
| NSFW | Content marked as adult or not safe for work. Hidden or restricted by default. |
| Platform-managed subscription | A subscription/follow relationship managed by a platform account. |
| Post | A content item in a community. |
| Post Detail | Screen showing full post content and comments. |
| Reddit compatibility | Version 1 target of working with Reddit-compatible content access. |
| Search | Direct lookup section for known content or communities. |
| Explore | Discovery section for browsing content or communities without a specific query. |
| Settings | Application configuration and secondary information. |
| Telemetry | Diagnostic or usage data collection. Antenna does not include this by default. |
| Analytics | Product usage measurement or behavioural event collection. Antenna does not include this by default. |
| Cache | Local temporary storage used to improve performance or reduce network use. |
| Restricted content | Content blocked by platform access, account, age, privacy, or policy restrictions. |
| Unsupported content | Content that exists but is not yet supported by Antenna. |
| ADR | Architecture Decision Record. A structured record of a significant decision. |

---

## Terminology rules

- Use **community** in generic documentation.
- Use **subreddit** only when specifically referring to Reddit terminology.
- Use **local favourites** for storage and architecture.
- Use **My Subs** for user-facing navigation if explanatory copy is present.
- Use **platform-managed subscriptions** for account-based subscriptions.
- Do not describe Antenna as official Reddit software.
- Do not use telemetry to describe ordinary content requests.
